# 数据源策略

## 1. 总原则

当前阶段优先轻资产验证，不把付费 API、真实账户、实盘能力作为前置条件。

## 2. 数据源优先级

1. P0：静态 CSV/JSON 历史数据回放。
2. P1：用户手动导出的 CSV/JSON。
3. P2：无需复杂认证的免费公共数据源。
4. P3：浏览器 DOM/截图/图表识别临时验证。
5. P4：正式券商 API。

## 3. 浏览器方案边界

浏览器方案只允许用于：

- 只读观察。
- 截图辅助分析。
- 页面导出文件。
- 临时验证数据可用性。

禁止用于：

- 真实下单。
- 长期生产数据源。
- 绕过 API 权限。
- 绕过风控。

## 4. Adapter 原则

所有数据源必须通过 adapter 进入系统。策略、风控、回测不得直接依赖某个浏览器页面、某个 API SDK 或某个导出格式。
